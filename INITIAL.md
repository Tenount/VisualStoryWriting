## FEATURE:

Visual Story-Writing: A system that automatically visualizes a story's chronological events, characters, and their actions and movements, allowing users to edit the story by manipulating these visual representations. The system uses GPT-4o to extract information from text and suggest edits.

Key features include:
- Visual timeline of story events that can be reordered to change the narrative sequence
- Interactive node-based representation of characters (entities) and locations
- Visual connection of characters to suggest edits reflecting new interactions
- Movement of character representations to suggest edits reflecting new positions
- Hover functionality to review chronology and visualize character movements
- Automatic text rewriting based on visual modifications
- Text editor with visual synchronization
- History tracking system to manage changes
- Support for studies and experiments with different writing approaches

The application provides both free-form writing mode and structured study modes for research purposes.

## OTHER CONSIDERATIONS:

- The system requires an OpenAI API key to function properly, as it relies heavily on GPT-4o for text analysis and generation
- The application was developed and tested primarily for recent versions of Google Chrome and Mozilla Firefox
- The system stores no user data locally; all processing happens in the browser with API calls to OpenAI
- Visual representations are generated by AI analysis of text, which may occasionally produce unexpected connections between entities or actions
- The application uses a complex state management system with multiple interconnected components that must stay synchronized
- The visual flow components require careful layout management to maintain readability as complexity increases
- Text synchronization between the editor and visual components requires careful diff handling to track changes
- Performance may degrade with very long texts or complex story structures with many interconnected elements
- The system includes both read-only study modes and fully interactive free-form modes
- Undo/redo functionality is implemented but interacts with AI-generated content which may require special handling
- The application includes logging functionality for research studies that should be considered when modifying core functionality
